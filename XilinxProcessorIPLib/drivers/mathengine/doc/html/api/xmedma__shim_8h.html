#define XMeDma_ShimChControl (   DmaInstPtr,
  ChNum,
  PauseStrm,
  PauseMm,
  Enable 
)
Value:
XMeGbl_Write32((DmaInstPtr->BaseAddress + \
ShimDmaCh[ChNum].CtrlOff), \
(XMe_SetField(PauseStrm, ShimDmaCh[ChNum].PzStr.Lsb, \
ShimDmaCh[ChNum].PzStr.Mask) | \
XMe_SetField(PauseMm, ShimDmaCh[ChNum].PzMem.Lsb, \
ShimDmaCh[ChNum].PzMem.Mask) | \
XMe_SetField(Enable, ShimDmaCh[ChNum].En.Lsb, \
ShimDmaCh[ChNum].En.Mask)))

Macro to configure the selected Shim DMA channel conrol bits.

Parameters
DmaInstPtr- Pointer to the Shim DMA instance.
ChNum- Channel number (0-S2MM0,1-S2MM1,2-MM2S0,3-MM2S1).
PauseStrm- When set, pauses the stream traffic.
PauseMm- When set, pauses the issuing of new AXI-MM commands.
Enable- Enable channel (1-Enable,0-Disable).
Returns
None.
Note
None.

u8 XMeDma_ShimPendingBdCount ( XMeDma_Shim *  DmaInstPtr,
u32  ChNum 
)

This API is used to get the count of scheduled BDs in pending.

Parameters
DmaInstPtr- Pointer to the Shim DMA instance.
ChNum- Should be one of XMEDMA_SHIM_CHNUM_S2MM0, XMEDMA_SHIM_CHNUM_S2MM1, XMEDMA_SHIM_CHNUM_MM2S0, or XMEDMA_SHIM_CHNUM_MM2S1.
Returns
Number of scheduled BDs in pending
Note
This function checks the number of pending BDs in the queue as well as if there's any BD that the channel is currently operating on. If multiple BDs are chained, it's counted as one BD.
